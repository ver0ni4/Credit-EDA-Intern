# Credit Risk EDA Case Study  
**Exploratory Data Analysis кредитных заявок для стажировки аналитика в Т-Банк**


## О проекте

Этот пет-проект создан с целью научиться работать с большими данными, попробовать сделать визуализацию и выводы по ней.

**Задача**: привести "грязные" банковские данные к удобному виду, провести разведочный анализ и выявить факторы, влияющие на дефолт клиентов.

**Датасет**: [Credit EDA Case Study](https://www.kaggle.com/datasets/venkatasubramanian/credit-eda-case-study) от Venkatasubramanian Sundaramahadevan  
- `application_data.csv` — текущие заявки клиентов (~307k строк, 122+ столбца)  
- `previous_application.csv` — история предыдущих заявок (~1.67M строк)  
- `columns_description.csv` — словарь переменных  

Данные имитируют реальные банковские процессы: много пропусков, аномалии, несбалансированный TARGET (~8% дефолтов).

**Бизнес-контекст**  
Цель — понять, кто склонен к дефолту, чтобы банк мог:  
- отказывать рискованным клиентам  
- снижать суммы кредитов  
- повышать ставки для риск-групп  
- не отклонять "хороших" заёмщиков  

## Что сделано в проекте

1. **Загрузка и первичный осмотр данных**  
2. **Очистка данных**  
   - Удалены столбцы с >50% пропусков  
   - Заполнены пропуски: медиана для числовых, мода/'Unknown' для категориальных  
   - Обработаны возможные аномалии  
3. **Нормализация структуры** 
   - Агрегация previous_application по клиентам 
   - Новые признаки: PREV_COUNT, PREV_APPROVED_PCT и др.  
4. **Exploratory Data Analysis**  
   - Распределение TARGET 
   - Визуализация дохода, истории одобрений vs дефолт
5. **Гипотезы и инсайты**  
   - Низкий доход -> выше риск дефолта  
   - Плохая кредитная история (низкий % одобрений в прошлом) -> сильный предиктор риска  

## Ключевые инсайты

- **Главные драйверы риска**:  
  - Клиенты с низким доходом имеют более высокий риск дефолта (медианный доход в группе TARGET=1 ниже).  
  - Низкий процент одобренных предыдущих заявок в истории — один из самых сильных сигналов просрочки.  
- Данные после очистки и агрегации стали удобными для моделирования .  
- Проект демонстрирует типичный пайплайн аналитика в банке: от сырых данных → чистая таблица -> бизнес-инсайты.
